## 1. Executive Summary

**ScoreMyPrompt**는 AI 프롬프트를 6개 차원(PROMPT: **P**recision, **R**ole, **O**utput format, **M**ission context, **S**tructure, **T**ailoring)으로 채점하고 개선 피드백을 제공하는 웹 서비스다.

- **문제:** AI 사용자의 90%+가 프롬프트를 체계적으로 작성하지 못하며, 결과물 품질이 들쭉날쭉
- **해결:** 5초 만에 프롬프트를 점수화하고, 구체적 개선 방향을 차원별로 제시
- **모델:** Freemium SaaS (무료 채점 + Pro $9.99/월)
- **핵심 무기:** $0/월 자동화 인프라 (N8N 패킹 비서 + 네이티브 예약 + 3-Tier AI)
- **목표:** 런칭 3개월 내 MRR $150+ 달성, 12개월 내 MAU 50,000+

### 7-1. 핵심 패러다임

**"N8N 패킹 비서 + 네이티브 예약 + 3-Tier AI"**

N8N은 SNS에 직접 발행하지 않는다. 콘텐츠를 "수집 → 초안 생성 → Google Sheets 정리 → 알림" 까지 자동 수행하고, 사람이 각 플랫폼의 내장 예약 기능(X 캘린더, LinkedIn 시계 아이콘, YouTube Studio 등)으로 $0에 게시한다.

**콘텐츠 리퍼포징 파이프라인:** 1개 블로그 포스트 → 트윗 3개 + LinkedIn 1개 + Shorts 스크립트 1개 + 뉴스레터 1단락. 이 파이프라인으로 1인 운영자가 주 5개 플랫폼에 일관된 콘텐츠를 유지한다.

### 8-2. 3-Tier AI 전략 (EcoBe AMAC N100 최적화)

EcoBe AMAC (Intel N100 4C/4T, 16GB LPDDR5, 512GB M.2 SSD, ~12-18W)에서 Ollama 8B는 메모리 부족으로 불안정하다. 따라서 3B 모델을 로컬 백업으로 두고 Groq 무료 API를 메인 LLM으로 활용한다.

| 비중 | 엔진 | 모델 | 용도 | 비용 |
|------|------|------|------|------|
| 55% | Groq 무료 API | Llama 3 70B (RPM 30) | 콘텐츠 초안 생성 (메인) | $0 |
| 30% | Ollama 로컬 | Llama 3.2 3B | 분류/태깅/짧은 요약 (백업) | $0 |
| 15% | Claude Max | Claude Opus/Sonnet | 고품질 장문 (수동 사용) | $0 (구독 내) |

**Why Groq > Ollama 8B on N100?**
1. **품질:** Groq Llama 70B >> 로컬 3B. 영어 마케팅 카피 품질 차이가 큼
2. **속도:** Groq는 세계 최고속 추론. N100 로컬은 8B 실사용 불가 수준
3. **안정성:** N100 16GB에서 8B 로드 시 Docker 컨테이너 전체 불안정
4. **백업:** Groq API 장애 시 Ollama 3B가 최소한의 태깅/요약 로컬 처리

**EcoBe AMAC Docker RAM 배분:**

| 컨테이너 | RAM | 역할 |
|----------|-----|------|
| N8N | ~512MB | 워크플로우 엔진 |
| PostgreSQL | ~256MB | N8N 데이터 |
| Redis | ~128MB | 캐시/큐 |
| Ollama 3B | ~4GB | 로컬 LLM |
| Cloudflare Tunnel | ~64MB | 외부 접근 |
| OS | ~1.5GB | Ubuntu |
| **여유** | **~9.5GB** | **안전 마진** |

> 제한: 16GB RAM 고정(온보드). Ollama는 3B까지만 안정 운영. 8B+ 모델 설치 금지.
